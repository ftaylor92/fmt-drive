XML Security Gateways


Purpose:

XML Security Gateways, which includes Web Services Security, are created to allow secure access to XML data. XML, and web services that use XML (like SOAP: Simple Object Access Protocol for passing XML data in a uniform way.), are popularly used over the web, so XML data security is necessary. XML Security Gateways allow only users that should to have access to XML data.


Implementation:

To create an XML security gateway/firewall to protect your data, a security design should be implemented with one or more products. Some solutions are proposed below under vendor-specific products, design documents and other new technologies. An industry standard way of implementing XML security is with, XML-Digital Signatures(DSIG)( http://www.w3.org/TR/xmldsig-core), XML encryption and X.509 certificates. This standard is called Web Services Security 1.0(WSS 1.0) and was approved by The Oasis international standards consortium(http://www.oasis-open.org).
